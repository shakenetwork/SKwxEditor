# 微信公众号文章采集编辑器

一键采集微信公众号文章，保留所有样式和图片！

---

## 🚀 功能特性

- ✅ **一键自动采集** - 通过PHP后端代理，无需手动复制粘贴
- ✅ **复制粘贴采集** - 纯前端方案，稳定兼容，零依赖
- ✅ **完整样式保留** - 保留文章的所有排版、图片和样式
- ✅ **批量采集** - 最多支持8篇文章
- ✅ **富文本编辑** - 使用UEditor编辑器，支持完整编辑功能
- ✅ **多种导出格式** - HTML、Markdown、JSON
- ✅ **PHP 5.6 兼容** - 完全支持PHP 5.6及以上版本

---

## 🎯 两种使用方式

### 方式一：PHP一键自动采集（推荐）

#### 📋 环境要求

- PHP 5.6+
- cURL扩展（必须启用）
- DOM扩展（必须启用）
- JSON扩展（必须启用）
- Web服务器（Apache/Nginx等）

#### 🛠️ 部署步骤

1. **上传文件到服务器**
   
   将以下文件上传到你的Web服务器：
   - `index.html` - 前端页面
   - `app.js` - 前端逻辑
   - `fetch_article.php` - PHP后端代理
   - `debug.php` - 调试与配置检查
   - 所有依赖的CSS/JS文件（UEditor、Tailwind等）

2. **检查配置**
   
   **重要：先访问 `debug.php` 检查你的环境！**
   
   这个页面会自动检查：
   - PHP版本
   - cURL扩展是否启用
   - DOM扩展是否启用
   - JSON扩展是否启用
   - 网络连通性

3. **确保PHP cURL扩展已启用**
   
   如果配置检查提示cURL未启用，按以下步骤操作：
   
   在php.ini中找到并修改：
   ```ini
   ;extension=curl  →  extension=curl
   ```
   
   去掉前面的分号，然后重启Web服务器。

4. **访问编辑器**
   
   在浏览器中访问 `index.html` 即可使用！

#### 💡 使用方法

1. 在输入框中粘贴微信公众号文章链接
2. 点击 **"一键采集"** 按钮
3. 等待几秒，文章自动添加到编辑器中！

#### ⚠️ 注意事项

- 如果PHP采集失败，可能是微信反爬虫机制导致，可以尝试其他方法
- 确保文章链接是公开可访问的
- 最多支持采集8篇文章
- 先访问 `debug.php` 确保环境配置正确

---

### 方式二：纯前端复制粘贴（稳定兼容）

#### 🚀 快速开始

1. **打开采集界面**
   - 直接在浏览器中打开 `index.html` 文件
   - 或者访问 http://localhost:3000 (如果本地服务器正在运行)

2. **访问目标文章**
   - 在新标签页打开你的微信公众号文章
   - 例如：`https://mp.weixin.qq.com/s/...`

3. **复制内容**
   - 在文章页面按 `Ctrl+A` 全选
   - 按 `Ctrl+C` 复制

4. **粘贴并解析**
   - 回到编辑器，点击 **"📋 粘贴内容并解析"**
   - 系统会自动提取标题、作者和内容

5. **保存文章**
   - 检查并编辑解析结果
   - 点击 **"✅ 保存文章到编辑器"**

#### 💡 技术特点

- **纯前端应用** - 不需要后端服务器
- **现代UI** - Tailwind CSS 构建
- **智能解析** - 自动识别微信公众号文章结构
- **零依赖部署** - 直接用浏览器打开即可

## 🔧 技术实现

### PHP后端代理 (fetch_article.php)

- 使用cURL模拟浏览器请求微信文章
- 设置合理的User-Agent和请求头
- 解析DOM提取标题、作者和正文内容
- 返回JSON格式数据给前端
- 完全兼容PHP 5.6

### 前端

- HTML5 + CSS3 (Tailwind CSS)
- JavaScript (ES6+)
- UEditor 富文本编辑器
- 本地存储 (localStorage)

## 📞 问题排查

### 常见问题

1. **JSON解析错误（PHP方式）**
   - 访问 `debug.php` 检查环境
   - 确保PHP版本 >= 5.6
   - 确保cURL/DOM/JSON扩展都已启用

2. **采集失败（PHP方式）**
   - 检查网络是否能访问微信
   - 尝试复制粘贴方式作为备选方案
   - 查看服务器日志了解具体错误

3. **cURL扩展未启用**
   - 编辑php.ini，取消 `extension=curl` 前面的注释
   - 重启Web服务器

4. **CORS跨域问题（纯前端方式）**
   - 由于浏览器安全限制，无法直接跨域抓取网页
   - 建议使用复制粘贴的工作流，这是最稳定和兼容的方式

### 检查清单（PHP方式）

- [ ] 访问 `debug.php` 检查配置
- [ ] PHP版本 >= 5.6
- [ ] cURL扩展已启用
- [ ] DOM扩展已启用
- [ ] JSON扩展已启用
- [ ] 服务器能访问外网
